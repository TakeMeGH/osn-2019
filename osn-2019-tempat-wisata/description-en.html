Description

There are M tourist attractions in Manado, numbered 1 to M. Today, there are N tourists in Manado, numbered 1 to N. Since there is a lot of tourist attractions, each tourist might only visit some of the tourist attractions today. The i-th tourist visits the A[i]-th, the (A[i] + 1)-th, the (A[i] + 2)-th, ..., and the B[i]-th tourist attractions.

A tourist will get acquainted with another tourist if there is at least a tourist attraction visited by both tourists. These tourists want to know the number of tourists they will get acquainted with, not including themselves.

Input Format

The input is given with the following format: 

N M
A[1] B[1]
A[2] B[2]
.
.
.
A[N] B[N]

Output Format

N lines: the i-th line contains the number of tourists got acquainted by the i-th tourist.

Explanation of Sample 1

In this sample, the first tourist will get acquainted with the second and third tourists.

Subtasks

For all subtasks:

  • 1 ≤ N ≤ 200,000.
  • 1 ≤ M ≤ 109.
  • 1 ≤ A[i] ≤ B[i] ≤ M.
  • N ≤ 200.
  • M ≤ 400.
  • N ≤ 2,000.
  • M ≤ 400,000.
  • There is no tourist attraction visited by more than two tourists.
  • No additional constraints.
